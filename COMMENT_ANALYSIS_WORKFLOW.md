# 爆文+评论分析工作流程

## 核心思路

**从爆文和评论中学习 → 提炼规则 → 创造新爆文**

---

## 完整工作流程

### 阶段 1: 分析爆文本身

**输入**:
- 爆文标题
- 爆文正文
- 数据指标（点赞数、收藏数、评论数）

**分析维度**:
- 标题钩子（数字、情绪词、悬念）
- 开头设计（痛点共鸣、场景代入）
- 内容结构（逻辑、节奏、信息密度）
- 结尾设计（行动号召、互动引导）
- 话题标签（热度、相关性）

**输出**:
- 提炼的规则（标题规则、结构规则、互动规则）

---

### 阶段 2: 分析热门评论（新增）

**输入**:
- 高赞评论（前20-50条）
- 评论的点赞数和回复数

**分析维度**:
1. **用户痛点识别**
   - 评论中提到的困扰和问题
   - 例：评论"我也是这样，每天加班到10点" → 痛点：加班严重

2. **用户需求挖掘**
   - 评论中询问的问题
   - 例：评论"有没有适合新手的方法？" → 需求：新手友好的方法

3. **情感共鸣点**
   - 评论中的情感表达
   - 例：评论"太真实了！说到心坎里了" → 共鸣点：真实感

4. **潜在选题发现**
   - 评论中提到的相关话题
   - 例：评论"时间管理也很重要" → 新选题：时间管理

5. **用户疑问收集**
   - 评论中的疑问和质疑
   - 例：评论"这个方法真的有效吗？" → 需要增加可信度证明

**输出**:
- 用户痛点列表
- 潜在选题列表
- 需要强化的内容点
- 新的互动策略

---

### 阶段 3: 综合分析和规则提炼

**综合考虑**:
- 爆文本身的成功要素
- 评论区反映的用户需求
- 用户的情感共鸣点
- 潜在的内容改进方向

**提炼规则**:
- 标题规则（基于爆文标题 + 评论反馈）
- 内容规则（基于爆文结构 + 用户需求）
- 互动规则（基于评论区互动模式）
- 选题规则（基于评论中的潜在话题）

**更新规则库**:
- 将新规则添加到 knowledge/rules/{platform}/base_rules.json
- 记录到 findings.md

---

### 阶段 4: 基于分析创作新内容

**输入**:
- 更新后的规则库
- 从评论中提炼的新选题
- 用户痛点和需求

**创作策略**:
1. **选题选择**: 从评论中发现的高需求话题
2. **痛点前置**: 在开头直击评论中提到的痛点
3. **需求满足**: 针对评论中的疑问提供解决方案
4. **可信度强化**: 针对评论中的质疑增加证明
5. **互动优化**: 根据评论区互动模式设计互动点

**输出**:
- 新的爆文内容
- 应用了最新规则
- 针对用户真实需求

## 评论分析的价值

### 1. 发现真实痛点

**爆文本身**: 展示了成功的表达方式
**评论区**: 揭示了用户的真实困扰

例：
- 爆文: "7天瘦10斤的方法"
- 评论: "我每天加班到10点，根本没时间运动"
- **发现**: 目标用户的核心痛点是"时间不够"，而不是"不知道方法"

### 2. 挖掘潜在选题

**评论中的高频问题 = 高需求选题**

例：
- 评论1: "有没有适合新手的方法？"
- 评论2: "零基础能做到吗？"
- 评论3: "新手应该从哪里开始？"
- **新选题**: "零基础新手的XXX入门指南"

### 3. 优化内容策略

**评论中的疑问 = 内容需要强化的点**

例：
- 评论: "这个方法真的有效吗？"
- **优化策略**: 增加科学依据、数据支撑、真实案例

### 4. 提升互动率

**评论中的互动模式 = 新的互动策略**

例：
- 高赞评论: "求详细步骤！"
- **互动策略**: 在正文中设置"评论区回复详细步骤"的钩子

## 总结

### 当前系统的适用性

✅ **完全适用**: ralph-loop + planning-with-files 可以用于爆文分析

✅ **可扩展**: 可以轻松扩展为"爆文+评论"综合分析

✅ **自动化**: 整个流程可以完全自动化执行

### 评论分析的优势

1. **发现真实需求**: 评论区是用户真实需求的宝库
2. **挖掘新选题**: 高频问题 = 高需求选题
3. **优化内容策略**: 针对性解决用户疑问
4. **提升互动率**: 基于评论区互动模式设计互动点
5. **持续学习**: 不断从用户反馈中学习和改进

### 建议的实现步骤

1. **立即可用**: 使用当前系统分析爆文本身
2. **短期扩展**: 在 Analyzer 中添加评论分析功能
3. **长期优化**: 建立评论数据库，进行大规模分析
